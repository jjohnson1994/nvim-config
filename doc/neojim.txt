1. INTRODUCTION                                       *neojim-introduction*

NeoJim is a modern Neovim configuration built for web development with React,
TypeScript, and modern JavaScript tooling. Uses Neovim 0.11+ native LSP APIs
with lazy loading and optimized performance.

99% Built by Claude Code

Full Documentation: https://yourusername.github.io/nvim/

==============================================================================
2. PHILOSOPHY                                             *neojim-philosophy*

NeoJim is designed with these core principles:

Zero Learning Curve ~
    All default Neovim keybindings and behaviors remain unchanged. If you
    already know vim, you can start using this config immediately without
    learning anything new.

Non-Conflicting ~
    Custom keybindings ONLY use the |<leader>| prefix (default: space) or
    non-standard keys. No default Neovim behaviors are overridden.

Gradual Enhancement ~
    Features are discoverable and can be adopted at your own pace. No forced
    workflows or opinionated patterns that must be learned upfront.

Lightweight Foundation ~
    This is a minimal opinionated layer on top of Neovim, not a complete IDE.
    It's a starting point for users to build upon, not an endpoint.

Openly Extensible ~
    The configuration is designed to be easy to customize, extend, and make
    your own. Complex abstractions and tightly coupled patterns are avoided.

3. PREREQUISITES                                   *neojim-prerequisites*

Required:
    • Neovim 0.11+ - Required for native LSP features
    • Git - For plugin management
    • Node.js & npm - For LSP servers and formatters
    • ripgrep - For fast searching with fzf-lua
    • fd - Recommended for file finding (required on macOS)
    • A Nerd Font - For icons (recommended: JetBrainsMono Nerd Font)

macOS Installation:
>
    brew install neovim ripgrep fd node
    brew install --cask font-jetbrains-mono-nerd-font
<

Linux (Ubuntu/Debian):
>
    sudo snap install nvim --classic
    sudo apt install git ripgrep fd-find nodejs npm
<

Windows (Scoop):
>
    scoop install neovim git ripgrep fd nodejs-lts
    scoop bucket add nerd-fonts
    scoop install JetBrainsMono-NF
<

4. INSTALLATION                                       *neojim-installation*

1. Backup existing configuration:
>
    mv ~/.config/nvim ~/.config/nvim.backup
<

2. Clone this configuration:
>
    git clone https://github.com/yourusername/yourrepo ~/.config/nvim
<

3. Launch Neovim (plugins will auto-install):
>
    nvim
<

4. Automatic installation:
    - Plugins install automatically (via lazy.nvim)
    - LSP servers, formatters, and linters auto-install after 3-second delay
      (via mason-tool-installer.nvim)

    Auto-installed tools:
    - LSP Servers: json-lsp, lua_ls, marksman, svelte, vtsls
    - Formatters: prettierd, stylua, eslint_d
    - Linters: markdownlint-cli2

5. Verify installation:
>
    :checkhealth lazy
    :checkhealth vim.lsp
<

8. CUSTOMIZATION                                   *neojim-customization*

CHANGE COLOR SCHEME ~
Edit lua/plugins/color-themes.lua:
>
    vim.cmd([[colorscheme tokyonight-night]])
    -- Change to one of:
    -- tokyonight-storm, tokyonight-day
    -- github_dark_dimmed, github_light
    -- catppuccin, kanagawa, onedark
    -- rose-pine, monokai-pro
<

ADD CUSTOM LSP SERVER ~
1. Create config file: lsp/your_server.lua
2. Add server settings:
>
    return {
      settings = {
        -- Your server settings
      },
    }
<
3. Add to ensure_installed in lua/plugins/lsp-config.lua

CHANGE LEADER KEY ~
Edit lua/options.lua:
>
    vim.g.mapleader = "\\"  -- Change to backslash
<

ENABLE INLAY HINTS ~
Uncomment in lua/options.lua:
>
    vim.lsp.inlay_hint.enable(true)
<
Or toggle with <Space>th

DISABLE AUTO-FORMAT ON SAVE ~
Toggle with <Space>uf or edit lua/plugins/conform.lua

TOGGLE PERSISTENCE ~
All UI toggles persist across Neovim sessions automatically:
• Diagnostic display mode (<Space>ud) - virtual_lines, virtual_text, or minimal
• Autoformat (<Space>uf) - enabled/disabled
• Auto-lint (<Space>ul) - enabled/disabled
• Auto-pairs (<Space>up) - enabled/disabled
• Inlay hints (<Space>th) - enabled/disabled
• Git blame (<Space>tb) - enabled/disabled

Settings are saved to: ~/.local/share/nvim/neojim_state.json

To reset all toggles to defaults:
>
    rm ~/.local/share/nvim/neojim_state.json
<

9. UPDATING                                             *neojim-updating*

AUTOMATIC UPDATES ~

NeoJim can automatically update everything on startup. Toggle with <leader>uu
(enabled by default).

When auto-update is enabled, the following are automatically updated:
• Config: Checks for new commits and notifies when available
• Plugins: Updates all lazy.nvim plugins in the background
• Treesitter parsers: Updates all parsers silently (only notifies if updates occur)
• Mason registry: Refreshes to check for package updates (doesn't auto-install)

All updates run asynchronously without blocking startup. The toggle state
persists across sessions.

Note: Mason packages are not auto-installed to avoid startup slowdowns.
Use :MasonUpdateAll to update all packages, or open :Mason and press 'U'.

MANUAL UPDATES ~

Config Updates:
>
    :UpdateConfig
<
This command will:
1. Check for uncommitted changes (warns if any exist)
2. Pull the latest changes from the remote repository
3. Notify you to restart Neovim to apply updates

Note: Make sure to commit or stash any local changes before updating to
avoid conflicts.

Plugin Updates:
>
    :Lazy sync
<

LSP Server and Tool Updates:
>
    :MasonUpdateAll   " Update all packages automatically
<
Or use the interactive UI:
>
    :Mason
    " Press 'U' to update all installed packages
<

10. TROUBLESHOOTING                               *neojim-troubleshooting*

PLUGINS NOT LOADING ~
>
    :Lazy sync
    :Lazy clean
    :Lazy update
<

LSP NOT WORKING ~
Check LSP is running:
>
    :lua vim.print(vim.lsp.get_clients({ bufnr = 0 }))
<
Check Mason installations:
>
    :Mason
<
Check health:
>
    :checkhealth vim.lsp
<

COMPLETION NOT SHOWING ~
Check blink.cmp is loaded:
>
    :lua print(vim.inspect(require("blink.cmp")))
<

FORMATTING NOT WORKING ~
1. Check formatters are installed via Mason
2. Verify file type is configured in lua/plugins/conform.lua
3. Manual format: <Space>cf

GIT SIGNS NOT SHOWING ~
1. Make sure you're in a git repository
2. Check gitsigns status:
>
    :Gitsigns toggle_signs
<
